# HomeStock AI Frontend Visual Direction

## Reference

Primary visual reference: `docs/design/homestock-visual-reference.png`

## Direction

HomeStock AI should feel like a warm household operating system, not a generic SaaS dashboard or a grocery marketplace. The interface should balance editorial warmth with practical approval-first workflows.

## Visual Principles

- Warm cream page background.
- Deep brown or charcoal typography.
- Large editorial headings for household identity and page context.
- Soft peach/tan cards for approval and action areas.
- Muted lavender blocks for summary metrics.
- Sage/deep green sections for memory, recurring items, and low-stock intelligence.
- Practical, scannable cards for grocery requests and cart items.
- Mobile-first bottom navigation.
- Clear, high-confidence approval actions.

## Interaction Priorities

- Keep grocery capture fast.
- Make pending, approved, cart-ready, and ordered states visually distinct.
- Make admin approval actions obvious without feeling aggressive.
- Keep mock provider uncertainty visible.
- Preserve existing backend/API/data contracts while improving presentation.

## Avoid

- Generic SaaS dashboard styling.
- Grocery marketplace visuals that imply live Swiggy ordering.
- Overly decorative gradients or abstract hero sections.
- Low-contrast beige-only palettes.
- Hidden approval or checkout actions.
